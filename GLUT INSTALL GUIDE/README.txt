▼2020年05月10日
・VisualStudio、システムへのGLUTインストールがなくてもビルド・デバッグできるよう仕込んだ。
(1)ランタイムファイル(glut32.dll)の仕込み
・プロジェクトフォルダ（上位のフォルダ）の中のDebugフォルダ。
・ここ実行可能ファイルができるので、これと同じ場所にglut32.dllをおいた。
(2)ライブラリファイル(glut32.lib)の仕込み
・ソリューションフォルダ（ソースファイルと同じ場所）にglut32.libをおいた。
＊プロパティ設定で「ライブラリフォルダ」をいじる必要はない。
(3)ヘッダファイル(glut.h)の仕込み
・ソリューションフォルダにヘッダファイルをローカルでインクルードできるよう
GLフォルダを作成し、この中に、ヘッダファイルの仕組みを組み込んだ。
・MacOSとの切り替えも行えるよう、glut.hという名称としながら、
OSによって、適切なヘッダファイルをインクルードするよう切り替えている。
(4)インクルードフォルダの設定
・先のステップで、ローカルフォルダに仕込んだヘッダファイルを読み込めるよう、
ソリューションのプロパティ設定で「インクルードフォルダ」にカレントフォルダとして
「.\」というフォルダを追記する設定を行った。
▼
これを整理するとこのフォルダの中の本README.txtと同じフォルダ内にある「GLUT Resources」の中の
ファイルやフォルダを以下のようにすれば、GLUTをシステムにインストールしなくても、
本実習のプログラムを使うことができる。
(1)glut32.dllをプロジェクトフォルダ>Debugにコピーする
(2)glut32.libをソリューションフォルダにコピーする
(3)GLフォルダをソリューションフォルダにコピーする
(4)ソリューションのプロパティ設定>インクルードフォルダを編集し「.\」を追加する
